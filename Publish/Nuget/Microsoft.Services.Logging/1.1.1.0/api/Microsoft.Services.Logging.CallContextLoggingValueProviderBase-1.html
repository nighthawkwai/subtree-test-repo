Class CallContextLoggingValueProviderBase<T>

This is a generic class that keeps a bunch of metadata for logging that shouldn't be passaround. Inheritors can have properties that are of primitive or serializeable types that will be passed safely between threads. The expectation is that the properties aren't null.

You can initialize a new context at any thread so that the threads following will have the new context.

Current

Gets or sets the current logging context. If one doesn't exist the default one is set as the current context and returned. When the value Current is set to null it reverts back to the original context.

Set semantics:

When the Current is set to null, the parent context that was present in the logical stack is returned. Otherwise the current provided value is a set the current logical context.

Remarks

The logging providers effectively form a rooted N-Ary tree of parent child relationships. When a provider is disposed, it is akin to a node in the tree being removed:

1. the current provider of the thread becomes the parent of the diposed
2. the disposed provider's children are added to the current provider's children.
3. the diposed provider is removed from the parent's children list.
